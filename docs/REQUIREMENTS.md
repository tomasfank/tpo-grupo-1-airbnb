# Requerimientos funcionales

## Gestión de usuarios
- El sistema debe permitir registrar usuarios como huésped o anfitrión.
- El sistema debe permitir consultar y actualizar el perfil de un usuario.
- El sistema debe permitir visualizar el perfil de un anfitrión junto con las propiedades que gestiona.

## Gestión de propiedades
- El sistema debe permitir a un anfitrión publicar propiedades con su información y servicios disponibles.
- El sistema debe permitir consultar propiedades según ubicación, tipo de propiedad y precio.
- El sistema debe permitir buscar propiedades por ciudad o mediante coordenadas geográficas.
- El sistema debe permitir a un anfitrión actualizar o eliminar sus propiedades.
- El sistema debe permitir visualizar todas las propiedades asociadas a un anfitrión.
- El sistema debe permitir definir la capacidad máxima de huéspedes por propiedad.

## Gestión de reservas
- El sistema debe permitir a un huésped realizar reservas sobre propiedades disponibles para un rango de fechas.
- El sistema debe permitir indicar la cantidad de huéspedes al momento de realizar la reserva.
- El sistema debe verificar la disponibilidad de fechas antes de confirmar una reserva.
- El sistema debe permitir consultar el estado de una reserva.
- El sistema debe permitir cancelar una reserva.

## Gestión de pagos
- El sistema debe registrar el pago asociado a una reserva, incluyendo monto, método de pago y estado.
- El sistema debe permitir consultar el estado del pago de una reserva.

## Gestión de reseñas
- El sistema debe permitir a un huésped dejar una reseña una vez finalizada la reserva.
- El sistema debe permitir consultar las reseñas de una propiedad.
- El sistema debe calcular y mostrar el promedio de calificaciones de una propiedad y de un anfitrión.
- El sistema debe garantizar que solo un huésped que haya realizado una reserva pueda dejar una reseña.
